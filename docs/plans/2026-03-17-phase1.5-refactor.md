## Task 1: Extract cmd_start helpers

**Files:** `src/main.rs`

`cmd_start` is ~150 lines with three nested concerns: replace handling, idempotent start, and sidecar spawn. Extract into:

1. `fn handle_replace(root, session_name) -> Result<()>` — the `if replace { ... }` block (lines 136-171). Handles orphan dir cleanup, force-kill of running session, wait for terminal + unlocked, remove_dir_all.

2. `fn try_idempotent_start(root, session_name, launch_spec) -> Result<Option<()>>` — the `AlreadyExists` match arm (lines 185-222). Returns `Ok(Some(()))` if idempotent return was printed, `Ok(None)` if orphan was cleaned and create should be retried, or `Err` for conflicts/terminal/starting states.

3. `fn spawn_and_wait_ready(session, launch_spec) -> Result<()>` — the pipe/spawn/readiness block (lines 225-282). Creates pipe, spawns sidecar, reads readiness signal, parses Meta, prints JSON, handles SpawnFailed exit code.

`cmd_start` becomes a ~40 line orchestrator:

```rust
fn cmd_start(name: &str, cmd: Vec<String>, stdin: bool, replace: bool, timeout: Option<u64>) -> anyhow::Result<()> {
    let session_name = SessionName::new(name)?;
    let root = SessionRoot::default_path()?;

    let mut launch_spec = LaunchSpec::new(cmd)?;
    launch_spec.stdin_mode = if stdin { StdinMode::Pipe } else { StdinMode::None };
    launch_spec.timeout_s = timeout;

    if replace {
        handle_replace(&root, &session_name)?;
    }

    let session = match session::create(&root, &session_name) {
        Ok(s) => s,
        Err(SessionError::AlreadyExists(_)) => {
            match try_idempotent_start(&root, &session_name, &launch_spec)? {
                Some(()) => return Ok(()), // idempotent return already printed
                None => session::create(&root, &session_name)?, // orphan cleaned, retry
            }
        }
        Err(e) => return Err(e.into()),
    };

    spawn_and_wait_ready(&session, &launch_spec)
}
```

## Task 3: Convert anyhow boundaries to .context()

**Files:** `src/main.rs`, `src/sidecar.rs`

Replace `map_err(|e| anyhow::anyhow!("msg: {e}"))` with `.context("msg")` at anyhow boundaries. This preserves the error source chain instead of flattening it.

**Pattern:**
```rust
// BEFORE
let spec_json = std::fs::read_to_string(&spec_path)
    .map_err(|e| anyhow::anyhow!("failed to read launch_spec.json: {e}"))?;

// AFTER
use anyhow::Context;
let spec_json = std::fs::read_to_string(&spec_path)
    .context("failed to read launch_spec.json")?;
```

**Scope:** Only `main.rs` and `sidecar.rs`. Leave `session.rs` and model code alone (they use typed `thiserror` errors correctly).

**Do NOT change:**
- `anyhow::bail!()` calls — these are terminal errors, not context wrapping
- `map_err` in typed library code (session.rs, model/)
- Cases where the map_err converts between error types (not just adding context)

Add `use anyhow::Context;` to both files.

**Verification:** `cargo test` — all 166 tests pass. No behavior change.

**Commit:** `refactor: convert anyhow error handling to .context() in CLI and sidecar`

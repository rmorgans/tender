### 1a: Change breadcrumb format in sidecar.rs

Currently at line 232:
```rust
let _ = std::fs::write(session_dir.join("child_pid"), child_pid.to_string());
```

Change to write JSON ProcessIdentity after we have the identity (move the write to after `process_identity` succeeds):

```rust
// After line 236 where child_identity is obtained:
let _ = std::fs::write(
    session_dir.join("child_pid"),
    serde_json::to_string(&child_identity).unwrap_or_default(),
);
```

Also update the error path (line 240) — the breadcrumb was written with bare PID before identity was known. Since we're moving the write to after identity, the error path just needs to NOT write the breadcrumb (child_pid file won't exist if we fail to get identity, which is fine — nothing to clean up since we kill+wait the child inline).

Remove the bare PID write at line 232. Add the identity write after line 236.

### 1b: Update cleanup_orphan_dir in main.rs

Replace the current logic that fabricates a bogus ProcessIdentity with `start_time_ns: 0`:

```rust
fn cleanup_orphan_dir(dir: &std::path::Path) {
    use tender::model::ids::ProcessIdentity;
    use tender::platform::unix as platform;

    let child_pid_path = dir.join("child_pid");
    if let Ok(content) = std::fs::read_to_string(&child_pid_path) {
        // Try JSON ProcessIdentity first (new format)
        if let Ok(identity) = serde_json::from_str::<ProcessIdentity>(&content) {
            match platform::process_status(&identity) {
                platform::ProcessStatus::AliveVerified => {
                    // Verified — safe to kill
                    let _ = platform::kill_process(&identity, true);
                }
                platform::ProcessStatus::Inaccessible => {
                    // Can't verify identity but process exists in different session.
                    // Attempt kill with degraded PID reuse safety (same as normal kill path).
                    let _ = platform::kill_process(&identity, true);
                }
                // Missing, IdentityMismatch, OsError — don't kill
                _ => {}
            }
        }
        // If it's old bare-PID format, we can't verify — skip kill (backwards compat)
    }
    let _ = std::fs::remove_dir_all(dir);
}
```

## Task 2: Generation increment on --replace

**Files:**
- Modify: `src/main.rs` (handle_replace writes generation file)
- Modify: `src/sidecar.rs` (reads generation file if present)

### 2a: Write generation file in handle_replace

In `handle_replace`, after reading the existing meta but before removing the dir, write the next generation to a temp location (or directly into the new session dir after creation). Simplest approach: write a `next_generation` file to the sessions root dir (not the session dir, since that gets deleted).

Better approach: pass generation through the sidecar command line or environment. But that changes the sidecar interface.

Simplest approach that doesn't change interfaces: after `remove_dir_all` in handle_replace, immediately create the new session dir and write a `generation` file:

Actually, the cleanest approach: `handle_replace` returns the next generation value. `cmd_start` passes it through to the sidecar via an environment variable or file.

Let's use a file — write `generation` to the session dir after `session::create` but before spawning the sidecar. The sidecar reads it if present, otherwise uses `Generation::first()`.

In `cmd_start`, after `handle_replace`:
```rust
fn handle_replace(...) -> anyhow::Result<Option<Generation>> {
    // ... existing logic ...
    // Before remove_dir_all, capture next generation
    let next_gen = existing_meta.generation().next();
    std::fs::remove_dir_all(existing.path())?;
    Ok(Some(next_gen))
}
```

Then in `cmd_start`, after creating the session dir:
```rust
if let Some(gen) = next_generation {
    std::fs::write(session.path().join("generation"), gen.as_u64().to_string())?;
}
```

In `sidecar.rs` `run_inner`, replace `Generation::first()` with:
```rust
let generation = {
    let gen_path = session_dir.join("generation");
    if let Ok(content) = std::fs::read_to_string(&gen_path) {
        let _ = std::fs::remove_file(&gen_path);
        content.trim().parse::<u64>()
            .ok()
            .and_then(|n| if n > 0 { Some(Generation::from_u64(n)) } else { None })
            .unwrap_or_else(Generation::first)
    } else {
        Generation::first()
    }
};
```

Note: `Generation` needs a `from_u64` constructor (or use the existing serde path). Check if one exists.

Looking at ids.rs: `Generation(u64)` has `first()` returning `Self(1)` and `next()` returning `Self(self.0 + 1)`. No `from_u64`. Add:
```rust
#[cfg(test)]
pub fn from_u64(n: u64) -> Self { Self(n) }
```

Wait — the sidecar needs it at runtime, not just tests. Make it a regular constructor:
```rust
/// Construct from a raw value. Returns first() if n is 0.
pub fn from_u64(n: u64) -> Self {
    if n == 0 { Self::first() } else { Self(n) }
}
```

# Prune — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** `tender prune` deletes terminal sessions older than a threshold. Filter-driven bulk operation only — no single-session delete, no orphan cleanup.

**Tech Stack:** Rust, clap, humantime, serde_json (NDJSON output).

**Quality gates:** `cargo clippy --all-targets` must pass. `cargo fmt` before each commit.

**Conventions:** All new tests use `SERIAL.lock()` guard and `TempDir` from existing test patterns. Tests that start sessions must kill/wait before returning.

---

## Per-Slice Invariant Table

| Invariant | Why it matters | Enforced by | Tested by | Known exceptions |
|-----------|---------------|-------------|-----------|-----------------|
| Locked sessions → skip `locked` | Sidecar may be mid-cleanup | `is_locked()` checked **first**, before meta read | `prune_skips_locked_session` | None |
| Non-terminal sessions → skip with status name | Live work must not be destroyed | `status.is_terminal()` after meta parse | `prune_skips_running_session` | None |
| Terminal but no `ended_at` → skip `missing_ended_at` | Conservative safety for model-impossible case | `status.ended_at().is_none()` check | `prune_skips_terminal_without_ended_at` | None |
| Missing meta.json → skip `missing_meta` | Orphan cleanup is separate concern | `meta_path.exists()` check before read | `prune_skips_missing_meta` | None |
| Corrupt meta.json → skip `corrupt_meta` | Bad JSON is not a delete signal | `read_meta()` parse failure | `prune_skips_corrupt_meta` | None |
| `--older-than` measures from `ended_at` | Age of completion, not creation | `now - ended_at.as_secs() > threshold` | `prune_respects_older_than` | None |
| `--older-than` required unless `--all` | Force intent on destructive command | Runtime check, early exit with error | `prune_without_filter_fails` | `--all` bypasses |
| `--dry-run` never calls `remove_dir_all` | Preview must be safe | Conditional on `dry_run` bool | `prune_dry_run_preserves_sessions` | None |
| `--namespace` scopes listing | No cross-namespace surprises | `session::list(root, namespace)` | `prune_respects_namespace` | None |
| Deletion is all-or-nothing per session | No partial cleanup | `fs::remove_dir_all(session.path())` | `prune_removes_entire_directory` | None |
| Deletion failure does not abort prune | One bad dir must not block the rest | `remove_dir_all` error → `action: "error"` line, continue | `prune_delete_failure_reports_error_and_continues` | None |
| Output: one NDJSON line per session in listing | Full visibility into every session considered | Line emitted for every `(ns, name)` from `session::list()` | `prune_output_format` | None |
| Summary includes `deleted`, `skipped`, `failed` | Machine-consumable totals, honest accounting | Separate counters, summary line at end | `prune_summary_counts_match` | None |
| Size accounting is best-effort, computed before delete | Failure to stat must not abort prune | `dir_size_bytes()` returns `Option<u64>` | `prune_size_error_does_not_abort` | None |

### CLI Contract

| Property | Value |
|----------|-------|
| Exit code 0 | Success (including 0 deletions) |
| Exit code 1 | Fatal error (can't read session root, invalid args) |
| Blocking | Yes — runs to completion |
| Idempotent | Yes — running twice with same filter is safe |
| Concurrent callers | Safe — locked sessions are skipped |
| Output format | NDJSON: one `type:session` line per session, one `type:summary` line at end |

### NDJSON Schema

**Delete (success):**
```json
{"type":"session","action":"delete","namespace":"ws-1","session":"claude-1","ended_at":1711500000,"reason":"ExitedOk","bytes":40960}
```

**Skip:**
```json
{"type":"session","action":"skip","namespace":"ws-1","session":"claude-2","skip_reason":"running"}
```

For `too_recent` skips, include `ended_at` for dry-run usefulness:
```json
{"type":"session","action":"skip","namespace":"ws-1","session":"claude-3","skip_reason":"too_recent","ended_at":1711590000}
```

Skip reasons: `locked`, `running`, `starting`, `missing_meta`, `corrupt_meta`, `missing_ended_at`, `too_recent`, `not_found` (race: directory disappeared between list and open)

**Error (deletion failed):**
```json
{"type":"session","action":"error","namespace":"ws-1","session":"claude-4","error":"permission denied"}
```

**Summary (always last line):**
```json
{"type":"summary","deleted":3,"skipped":1,"failed":1,"bytes_reclaimed":122880,"dry_run":false,"namespace":null}
```

`namespace` is the filter namespace if `--namespace` was provided, `null` otherwise.

### Task 2: Add `Prune` variant to CLI enum and dispatch

Add to `Commands` enum in `main.rs`:

```rust
/// Delete terminal sessions older than a threshold
Prune {
    /// Delete sessions ended more than DURATION ago (e.g. 7d, 24h, 30m)
    #[arg(long, value_parser = parse_duration)]
    older_than: Option<std::time::Duration>,
    /// Delete all terminal sessions regardless of age
    #[arg(long)]
    all: bool,
    /// Namespace to prune (prunes all namespaces if omitted)
    #[arg(long)]
    namespace: Option<String>,
    /// Show what would be deleted without deleting
    #[arg(long)]
    dry_run: bool,
},
```

Add a `parse_duration` wrapper function that calls `humantime::parse_duration`. This keeps the dependency behind a thin boundary.

Add dispatch in `main()`:
```rust
Commands::Prune { older_than, all, namespace, dry_run } => {
    // Runtime validation: older_than required unless all
    match namespace
        .map(|s| Namespace::new(&s).map_err(anyhow::Error::from))
        .transpose()
    {
        Ok(ns) => commands::cmd_prune(older_than, all, ns.as_ref(), dry_run),
        Err(e) => Err(e),
    }
}
```

Note: `--namespace` for prune follows the `list`/`watch` pattern (optional, filters to one namespace) not the per-session pattern (required for session identity). So use `Option<&Namespace>`, not `resolve_namespace()`.

### Task 3: Add `commands/prune.rs` — core logic

**Function signature:**
```rust
pub fn cmd_prune(
    older_than: Option<Duration>,
    all: bool,
    namespace: Option<&Namespace>,
    dry_run: bool,
) -> anyhow::Result<()>
```

**Validation (top of function):**
```rust
if older_than.is_none() && !all {
    anyhow::bail!("either --older-than or --all is required");
}
```

**Core loop — for each `(ns, name)` from `session::list()`:**

Classification is structural, not string-based. Do not match on `SessionError::Corrupt` reason text.

```
1. Build session dir path: root/<ns>/<name>/
2. open_raw(root, &ns, &name) — if NotFound, skip (shouldn't happen since list found it, but race-safe)
3. Check is_locked(&session_dir) — if locked, emit skip "locked", continue
4. Check meta_path.exists() — if false, emit skip "missing_meta", continue
5. read_meta(&session_dir) — if Err, emit skip "corrupt_meta", continue
6. Check status.is_terminal() — if false, emit skip with status name ("running"/"starting"), continue
7. Check status.ended_at() — if None, emit skip "missing_ended_at", continue
8. If older_than is Some, check age: now - ended_at.as_secs() — if < threshold, emit skip "too_recent" with ended_at, continue
9. Compute dir_size_bytes(session_dir.path()) — Option<u64>, None on error
10. If dry_run: emit delete line (with bytes), do NOT delete
11. If not dry_run: attempt fs::remove_dir_all(session_dir.path())
    - On success: emit delete line (with bytes)
    - On failure: emit error line with error message, increment failed counter, continue
12. Increment appropriate counter (deleted/skipped/failed)
```

**Counters:** `deleted: u64`, `skipped: u64`, `failed: u64`, `bytes_reclaimed: u64`

After loop, emit summary line.

**NDJSON output:** Use `serde_json::to_string()` on typed structs, then `println!()`. Define output structs within `prune.rs` — they are output-only, not shared model types.

**Output structs:**

```rust
#[derive(Serialize)]
#[serde(tag = "type")]
#[serde(rename_all = "snake_case")]
enum PruneOutput {
    Session(SessionAction),
    Summary(PruneSummary),
}

#[derive(Serialize)]
struct SessionAction {
    action: &'static str,  // "delete", "skip", "error"
    namespace: String,
    session: String,
    #[serde(skip_serializing_if = "Option::is_none")]
    ended_at: Option<u64>,
    #[serde(skip_serializing_if = "Option::is_none")]
    reason: Option<String>,        // exit reason for deletes
    #[serde(skip_serializing_if = "Option::is_none")]
    skip_reason: Option<&'static str>,  // for skips
    #[serde(skip_serializing_if = "Option::is_none")]
    error: Option<String>,         // for errors
    #[serde(skip_serializing_if = "Option::is_none")]
    bytes: Option<u64>,            // for deletes
}

#[derive(Serialize)]
struct PruneSummary {
    deleted: u64,
    skipped: u64,
    failed: u64,
    bytes_reclaimed: u64,
    dry_run: bool,
    namespace: Option<String>,
}
```

**Files:** `src/commands/prune.rs`

### Task 5: Helper `fn dir_size_bytes(path: &Path) -> Option<u64>`

Place in `commands/prune.rs` (private helper, not shared).

Recursive walk via `fs::read_dir()` + `entry.metadata().len()`. Follows symlinks for size (metadata, not symlink_metadata) — session dirs should not contain symlinks, but if they do, counting the target size is more honest.

Returns `None` on any IO error at any level — never panics, never aborts prune. Accumulated size is best-effort: if one file fails to stat, the total is still returned for the files that succeeded (not `None`). Only return `None` if the top-level `read_dir` fails.

Computed **before** delete. The delete line reports the size that was computed before removal.

**Files:** `src/commands/prune.rs`

### Task 6: Integration tests

All tests in `tests/cli_prune.rs`. Use `assert_cmd` and `predicates` for CLI testing. Use `TempDir` as `HOME` override to isolate session storage.

**Test helper:** Create a terminated session on disk by:
1. `tender start <name> --namespace <ns> -- sleep 0` (exits immediately)
2. `tender wait <name> --namespace <ns>` (ensure terminal)
3. Optionally backdate `ended_at` by rewriting meta.json

**Tests:**

1. **`prune_deletes_terminal_session`**
   - Start + wait a session, run `prune --all`, verify directory gone and output shows `action: delete`

2. **`prune_skips_running_session`**
   - Start `sleep 60`, run `prune --all`, verify skip with `running`, kill session after

3. **`prune_respects_older_than`**
   - Create two terminal sessions. Backdate one's `ended_at` to 8 days ago. Run `prune --older-than 7d`. Only the old one is deleted.

4. **`prune_dry_run_preserves_sessions`**
   - Create terminal session, run `prune --all --dry-run`, verify directory still exists and output shows `action: delete`

5. **`prune_without_filter_fails`**
   - Run `tender prune` with no flags, assert exit code 1 and error message

6. **`prune_skips_corrupt_meta`**
   - Create session dir with garbage in meta.json, run `prune --all`, verify skip with `corrupt_meta`

7. **`prune_skips_missing_meta`**
   - Create session dir with no meta.json (just mkdir), run `prune --all`, verify skip with `missing_meta`

8. **`prune_respects_namespace`**
   - Create terminal sessions in two namespaces, prune one, verify the other is untouched

9. **`prune_output_format`**
   - Parse NDJSON output, verify each line is valid JSON with correct `type` field, verify summary line is last

10. **`prune_summary_counts_match`**
    - Mix of deletable and skippable sessions, verify summary `deleted` + `skipped` + `failed` equals total sessions listed

11. **`prune_delete_failure_reports_error_and_continues`**
    - Create two terminal sessions, make one's directory unreadable (`chmod 000` on a file inside, or use a platform-appropriate trick), run `prune --all`, verify one error + one delete + `failed: 1` in summary
    - Note: this test may be platform-sensitive. If permission tricks are unreliable in CI, mark `#[ignore]` with a comment explaining why.

**Files:** `tests/cli_prune.rs`

# Wrap — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** `tender wrap` runs a command, captures its stdin/stdout/stderr/exit code, and appends an annotation record to the session's `output.log`. This enables transparent hook event ingestion for cmux and other agent frameworks.

**Scope:** wrap only. No exec, no sentinel framing, no streaming stdin tee.

**Tech Stack:** Rust, clap, serde_json, std::process::Command, signal handling via `AtomicBool`.

**Quality gates:** `cargo clippy --all-targets` must pass. `cargo fmt` before each commit.

**Conventions:** All new tests use `SERIAL.lock()` guard and `TempDir`. Tests that start sessions must kill/wait before returning.

---

## Design Decisions (locked)

| # | Decision | Choice |
|---|----------|--------|
| 0 | Run targeting | Sidecar injects `TENDER_RUN_ID` (+ session, namespace, session_dir) into child env at spawn. Wrap reads `TENDER_RUN_ID` from its own environment. |
| 1 | Storage | Append `A`-tagged line to session's `output.log` |
| 2 | Size limit | ≤4096 bytes per annotation line. Truncate payload fields if exceeded. |
| 3 | Stdin model | Buffered: read all stdin, then pipe to child. No concurrent tee. |
| 4 | Signal forwarding | On SIGTERM: set flag → forward to child → wait 5s → SIGKILL → write annotation → exit |
| 5 | Concurrent wraps | Direct `O_APPEND` writes. Pragmatic local-FS assumption, not a formal atomicity guarantee. Best-effort. |

## Annotation Line Format

Same format as existing output.log lines:

```
{epoch_secs}.{micros:06} A {single-line-json}\n
```

JSON payload:

```json
{"source":"cmux.claude-hook","event":"pre-tool-use","run_id":"019...","data":{"hook_stdin":{},"hook_stdout":{},"hook_stderr":"","hook_exit_code":0,"command":["cmux","claude-hook","pre-tool-use"],"truncated":false}}
```

Total line length (timestamp + space + tag + space + json + newline) must be ≤4096 bytes.

If payload exceeds limit after serialization:
1. Truncate `hook_stdin`, `hook_stdout`, `hook_stderr` fields to fit
2. Set `truncated: true`
3. If still too large after truncating all three to empty, drop the annotation entirely and write to stderr

## Per-Slice Invariant Table

| Invariant | Why it matters | Enforced by | Tested by | Known exceptions |
|-----------|---------------|-------------|-----------|-----------------|
| Wrap exits with child's exit code | Caller must see real result | `std::process::exit(code)` | `wrap_preserves_child_exit_code` | Signal death → 128+signum |
| Stdin fully consumed before child spawn | Hook protocol sends complete JSON blob | `stdin.read_to_end()` before `Command::new().spawn()` | `wrap_passes_stdin_to_child` | None |
| Annotation is single-line JSON | output.log is newline-delimited | `serde_json::to_string()` (compact, no newlines) | `wrap_writes_annotation_line` | None |
| Annotation line ≤4096 bytes | Pragmatic append atomicity on local FS | Size check + truncation before write | `wrap_truncates_large_payload` | None |
| `TENDER_RUN_ID` required | Attribution must be explicit, no guessing | Read from env, fail if missing | `wrap_fails_without_run_id` | None |
| `--source` validated, `tender.*` reserved | Source namespace integrity | `Source::new()` rejects reserved prefix | `source_rejects_tender_prefix` | None |
| Child stdout/stderr captured then replayed | Annotation records actual output; caller sees same bytes after child exits | `child.wait_with_output()` then `io::stdout().write_all()` | `wrap_captures_and_replays_stdout` | Output appears after child exit, not streamed live |
| SIGTERM forwarded to child | No orphaned processes | Signal flag + forward in main flow | `wrap_forwards_sigterm` (unix only) | Windows: not implemented in v0 |
| Annotation written even if child is killed | Best-effort observability | Write attempt after child wait completes | `wrap_annotates_killed_child` | Wrap itself killed before write |
| Session dir resolved structurally | Env-inherited path could be stale or mismatched | `SessionRoot + namespace + session` path construction via `session::open()` | `wrap_fails_for_nonexistent_session` | None |

### CLI Contract

| Property | Value |
|----------|-------|
| Exit code | Mirrors child's exit code. 1 if wrap itself fails before spawn. |
| Blocking | Yes — runs child to completion |
| Idempotent | N/A — each invocation is unique |
| Concurrent callers | Safe — direct append, no locking |
| Output | Child stdout/stderr captured, then replayed to wrap's stdout/stderr after child exits. Annotation goes to output.log, not stdout. |

## Tasks

### Task 0: Inject TENDER_* env vars into supervised child

**Prerequisite for wrap.** The sidecar must export `TENDER_SESSION`, `TENDER_NAMESPACE`, `TENDER_RUN_ID`, `TENDER_GENERATION`, and `TENDER_SESSION_DIR` into the child process environment at spawn time.

Currently these are only set for on-exit callbacks (sidecar.rs:332-337). The child itself doesn't see them.

**Implementation:**

In `sidecar.rs::run_inner()`, after meta is created and before `spawn_child()`:

1. Build a map of TENDER_* vars from the meta
2. Build an effective env map: start with `LaunchSpec.env` (user-supplied), then overlay TENDER_* vars on top
3. TENDER_* vars are authoritative — they override any user-supplied values with the same keys

This means the child, and every process it spawns (including hook commands), inherits the correct TENDER_RUN_ID even if the user accidentally set a conflicting env var.

**Important:** Do not mutate `LaunchSpec.env` — that would change the canonical hash and break idempotent start. Build a separate effective env map for the spawn call.

**Test:** `wrap_child_inherits_tender_env` — start a session running `env`, grep output.log for TENDER_RUN_ID.

**Files:** `src/sidecar.rs`, `tests/cli_wrap.rs`

### Task 3: Add `Wrap` CLI variant and dispatch

```rust
/// Run a command and record an annotation in the session's event log
Wrap {
    /// Session name
    #[arg(long)]
    session: String,
    /// Namespace for session grouping
    #[arg(long)]
    namespace: Option<String>,
    /// Annotation source (e.g. "cmux.claude-hook")
    #[arg(long)]
    source: String,
    /// Event name (e.g. "pre-tool-use")
    #[arg(long)]
    event: String,
    /// Command and arguments
    #[arg(trailing_var_arg = true, required = true)]
    cmd: Vec<String>,
},
```

**Run ID resolution:** Wrap reads `TENDER_RUN_ID` from its own environment. If not set, fail with a clear error message.

**Namespace/session resolution:** Wrap can also read `TENDER_SESSION` and `TENDER_NAMESPACE` from env as defaults, but explicit `--session` and `--namespace` flags override. This means when wrap is called inside a hook chain, the user doesn't need to pass `--session` and `--namespace` — they're inherited.

Update: make `--session` optional, default from `TENDER_SESSION` env. Same for `--namespace` defaulting from `TENDER_NAMESPACE` env. Fail if neither flag nor env is available.

### Task 4: Implement `commands/wrap.rs` — core logic

**Function signature:**
```rust
pub fn cmd_wrap(
    session: &str,
    namespace: &Namespace,
    source: &Source,
    event: &str,
    cmd: Vec<String>,
) -> anyhow::Result<()>
```

**Core flow:**

```
1. Read TENDER_RUN_ID from env → fail if missing
2. Resolve session dir structurally: SessionRoot + namespace + session → path
   (Do NOT trust TENDER_SESSION_DIR from env as source of truth)
3. Verify session dir exists via session::open()
4. Read all stdin into buffer (stdin().read_to_end())
5. Install SIGTERM handler (set AtomicBool flag)
6. Spawn child: Command::new(cmd[0]).args(&cmd[1..])
   - stdin: piped — write buffered stdin, then drop handle (child sees EOF)
   - stdout: piped — captured into Vec<u8>
   - stderr: piped — captured into Vec<u8>
7. Wait for child exit (wait_with_output or thread-based capture)
   - If SIGTERM flag set during wait: forward SIGTERM to child, wait 5s, SIGKILL if still alive
8. Replay captured stdout to wrap's stdout, captured stderr to wrap's stderr
9. Build annotation payload JSON (includes captured stdout/stderr/exit code)
10. Truncate if needed to fit 4096-byte line limit
11. Append A-tagged line to output.log (OpenOptions::append + O_APPEND)
12. Exit with child's exit code
```

**Stdout/stderr contract:** Child stdout and stderr are piped and captured into buffers. After the child exits, wrap replays the captured bytes to its own stdout/stderr. This means output appears after child exit, not streamed live. This is correct for the hook use case where commands are short-lived and callers consume output after completion. The annotation records what the hook actually produced.

**Signal handling (Unix):**
```rust
static SIGTERM_RECEIVED: AtomicBool = AtomicBool::new(false);

// Before spawning child:
unsafe {
    libc::signal(libc::SIGTERM, sigterm_handler as libc::sighandler_t);
}

extern "C" fn sigterm_handler(_: libc::c_int) {
    SIGTERM_RECEIVED.store(true, Ordering::SeqCst);
}
```

After child wait returns (or periodically if using a polling wait):
```rust
if SIGTERM_RECEIVED.load(Ordering::SeqCst) {
    // Forward SIGTERM to child, wait with timeout, SIGKILL if needed
}
```

**Windows:** No signal forwarding in v0. Document as known limitation.

**Annotation write:**
```rust
fn write_annotation(session_dir: &Path, payload_json: &str) -> io::Result<()> {
    let ts = timestamp_micros(); // same format as sidecar
    let line = format!("{ts} A {payload_json}\n");
    assert!(line.len() <= 4096, "annotation line exceeds PIPE_BUF");

    let log_path = session_dir.join("output.log");
    let mut file = OpenOptions::new()
        .create(true)
        .append(true)
        .open(&log_path)?;
    file.write_all(line.as_bytes())?;
    Ok(())
}
```

### Task 6: Extend watch to emit annotation events

In `commands/watch.rs`, the log-tailing loop currently only processes `O` and `E` tags (via `LogLine::parse`). After Task 2, `A` lines will also parse.

**Changes:**

1. Add `--annotations` flag to watch CLI (main.rs)
2. In the log-tailing section of watch, when a parsed line has tag `A`:
   - Parse the JSON content to extract `source` and `event` fields
   - Emit as `kind: "annotation"`, `name: "annotation.<event>"`, with the full JSON as data
   - Use the `source` from the annotation payload (not hardcoded `tender.sidecar`)
3. Default behavior: `--annotations` is opt-in. If neither `--events`, `--logs`, nor `--annotations` is specified, emit events + logs (existing default). Annotations require explicit `--annotations` flag.
4. `--annotations` can combine with `--events` and/or `--logs`.

**Watch event for an annotation:**
```json
{"ts":1711500000.234567,"namespace":"ws-1","session":"claude-1","run_id":"019...","source":"cmux.claude-hook","kind":"annotation","name":"annotation.pre-tool-use","data":{"hook_stdin":{},"hook_stdout":{},"hook_stderr":"","hook_exit_code":0,"command":["cmux","claude-hook","pre-tool-use"],"truncated":false}}
```

**Files:** `src/main.rs` (--annotations flag), `src/commands/watch.rs`

### Task 7: Integration tests

All tests in `tests/cli_wrap.rs`.

**Test helpers:**
- `create_running_session(root, name, namespace)` — start a `sleep 60` session, wait for Running
- `create_terminal_session(root, name, namespace)` — start `echo done`, wait for terminal
- Cleanup: kill running sessions after each test

**Tests:**

1. **`wrap_child_inherits_tender_env`**
   Start a session running `env`, wait for terminal, grep output.log for TENDER_RUN_ID, TENDER_SESSION, TENDER_NAMESPACE.

2. **`wrap_preserves_child_exit_code`**
   Run `tender wrap --session s --source test.src --event test -- sh -c "exit 42"` with TENDER_RUN_ID set. Assert wrap exits with code 42.

3. **`wrap_captures_and_replays_stdout`**
   Run `tender wrap ... -- echo hello`, verify wrap's stdout contains "hello" and the annotation payload contains `hook_stdout` with "hello".

4. **`wrap_writes_annotation_line`**
   Run wrap against a running session, then read output.log and verify an `A`-tagged line exists with correct source and event.

5. **`wrap_fails_without_run_id`**
   Run wrap without TENDER_RUN_ID env var, assert exit code 1 and error message.

6. **`wrap_fails_for_nonexistent_session`**
   Run wrap with --session pointing to a nonexistent session, assert failure.

7. **`source_rejects_tender_prefix`**
   Run wrap with `--source tender.foo`, assert exit code non-zero.

8. **`wrap_annotation_visible_in_watch`**
   Start a session, run wrap to emit an annotation, run `tender watch --namespace <ns> --annotations --from-now` (with timeout), verify the annotation event appears in the NDJSON stream.

9. **`wrap_defaults_session_from_env`**
   Set TENDER_SESSION and TENDER_NAMESPACE env vars, run wrap without --session/--namespace flags, verify it works.

10. **`wrap_truncates_large_payload`**
    Pipe >4096 bytes of stdin to wrap via `cat`, verify annotation line in output.log is ≤4096 bytes and `truncated: true`.
